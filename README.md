# Микросервис для сокращения ссылок

Тестовое задание — сервис для сокращения URL, написанный на FastAPI.

## Что  внутри

- Короткий идентификатор генерируется через **Base62** — алгоритм написан с нуля в `app/utils.py`
- Race condition при обновлении счётчика переходов решён атомарным инкрементом через F-выражения SQLAlchemy (`app/crud.py`)
- Код разбит по слоям: Schemas, Models, CRUD, Utils — чтобы не превращалось в кашу
- Входящие URL валидируются через `HttpUrl` в Pydantic, так что мусор не пройдёт
- Если ссылка не найдена — возвращается нормальный JSON с `404`, а не просто пустой ответ

## Стек

`Python 3.12+`, `FastAPI`, `SQLAlchemy 2.0 (async)`, `PostgreSQL`, `Pytest`, `Docker Compose`, `pydantic-settings`

---

## Запуск

1. Склонируйте репозиторий и перейдите в папку с проектом
2. Скопируйте файл с настройками:
```bash
cp .env.example .env
```
> Дефолтные значения уже настроены под Docker, так что менять ничего не нужно

3. Соберите и запустите контейнеры:
```bash
docker-compose up --build -d
```

4. После запуска сервис будет доступен по адресам:
   - Swagger UI: http://localhost:8000/docs
   - API: http://localhost:8000/

## Тесты

Тесты работают через `pytest`, используют отдельные таблицы и очищают их перед каждым запуском — чтобы не мешать друг другу.

Запускать удобнее прямо внутри контейнера:

```bash
docker-compose exec app pytest -v
```

Покрыты основные сценарии: создание ссылки, редирект, инкремент счётчика и запрос несуществующего кода.
